📄 Aviation Checklist Generator — Help

Page Layout

The generator produces a single A4 landscape page that is designed to be folded vertically in the centre. The left half contains normal (pre-flight / flight) procedures, the right half contains emergency procedures.

ParameterWhat it controlsDefault
outer_marginWhite space around the entire page (all four edges)8 mm
fold_marginExtra space on each side of the centre fold line so content isn't cut when folding8 mm
col_gapGap between adjacent columns on the same half3 mm
columns per half2 → 4 total columns (2 normal + 2 emergency) · 3 → 6 total3 / half
scaleMultiplier applied to all font sizes and bar heights (0.8 = smaller, 1.2 = larger)1.0
line_spacingMultiplier for the vertical space between checklist items1.35

Item Styles

StyleAppearanceTypical use
(none)Label left · callout right · dot leaderStandard checklist item
blueBlue italic, both label and calloutEngine start sequence steps
noteSmall grey italic, indented, no calloutSub-notes, limits, tolerances
warnRed bold italic, full widthCritical warnings, EVACUATE
centeredCentred black boldPhase dividers (— ON RWY —)
centered_blueCentred blue boldSub-phase markers (ABM THRESHOLD)
redRed bold, label + callout with dot leaderCritical speeds (Vne, glide)

Speeds section — use type: speeds on a section to render it as a compact two-column table (no dot leaders). Add body_bg: true for a light grey background. Individual speed items accept style: red to highlight critical values (Vne, best glide, etc.).

Section Header Colours

Normal-procedure sections accept a header_color key. Emergency sections are always red. Speeds sections are always blue.

ValueAppearanceSuggested use
blackBlack background, white text (default)Before Start, Parking, Before T/O
blueDark blue (#2F5496), white textEngine Start, Run Up, Approach
greenMid-tone green (#4E7A3A), white textTaxi, Cruise, After Landing
yellowDark amber (#B8860B), white textParking, After Landing

YAML Structure

Every checklist file has three top-level keys: meta, normal, and emergency.

normal sections go on the left half; emergency sections go on the right half. Columns are numbered 1..N within each half independently.

# ── Aircraft identity ─────────────────────────
meta:
  callsign:        "D-EADX"
  icao_type:       "DA40"
  model:           "Diamond DA40 D TDI"
  revision:        "Rev 1  |  2024-11"
  # Layout — settable here or via the sidebar sliders
  columns:         6  # 4 or 6  (= 2 or 3 cols per half)
  font:            dejavu-condensed
  scale:           1.1  # font size multiplier
  line_spacing:    1.5
  fold_margin_mm:  8
  col_gap_mm:      3.0
  outer_margin_mm: 8

# ── Normal procedures (LEFT half) ─────────────
normal:
  - col: 1
    sections:
      - title: "BEFORE START"
        header_color: black  # black | blue | green | yellow
        body_bg: false  # true = light grey background
        items:
          - Parking Brake: SET
          - AVIONIC MASTER: "OFF"  # quote ON/OFF — YAML reads them as booleans!
          - Mixture: FULL RICH
            style: blue  # blue | note | warn | centered | centered_blue | red
          - Wait for glow indicator:  # item with no callout
            style: note

      - title: "OPERATING SPEEDS [KIAS]"
        type: speeds  # two-column table, no dot leaders
        body_bg: true  # recommended: light grey background
        items:
          - Vr: 59
          - Vy: 78
          - Vne: 178
            style: red  # red = highlighted speed (Vne, glide, etc.)

# ── Emergency procedures (RIGHT half) ─────────
emergency:
  - col: 1  # col 1 of emergency = col 4 on the page
    sections:
      - title: "ENGINE FAILURE AFTER T/O"
        items:
          - Glide: ESTABLISH
          - ENGINE MASTER: "OFF"
          - Mayday: TRANSMIT
            style: warn
          - EVACUATE:
            style: warn
